# 00_Overview — プロジェクト概要シート

## 1. プロダクト名
- Tsumami Designer MVP（つまみ細工デザインアプリ）

## 2. 目的
- つまみ細工（丸つまみ／剣つまみ）の花を、**花（Flower）× レイヤー（Layer）× 花びら（Petal）**で設計し、
  - SVGプレビューで形状・色・配置を確認
  - 色別枚数と工程（レイヤー順）を **Recipe** として出力
  - JSONで保存・共有（手渡し）できる

## 3. 想定ユーザー
- つまみ細工の制作者（趣味〜準プロ）
- 試作段階で配色・段数・枚数・配置を検討したい人

## 4. MVPスコープ
### In
- 画面：Home / New / Editor / Recipe
- ローカル保存（localStorage）
- JSON Import/Export
- Undo/Redo（スナップショット）
- 花（Flower）単位の Copy/Paste
- 花（Flower）同士の描画順制御（Front/Back/+1/-1）
- レイヤーを外側へ追加（外側踏襲＋角度ずらし）
- ツリー展開状態の永続化

### Out（今後）
- サーバ保存／アカウント
- SVG/PNG/PDF出力の整備
- 花びら単位のプロパティ編集UI（データは草案あり）
- 細かな物理寸法（布厚・折り代）シミュレーション

## 5. 主要要件（抜粋）
- **デフォルト花びら枚数 = 6**（初期起動時・花追加時）
- レイヤー追加は **最外周に追加**し、外側レイヤーを踏襲
- レイヤー追加時は **外側レイヤーと角度を半ピッチずらす**（内側の隙間に見える）
- Layers（左）一覧はスクロール可能
- Flower→Layer の階層表示。展開/折りたたみが可能で、状態は再起動後も維持
- 位置やスケールは **絶対値**表記（Properties）

## 6. 技術方針
- React 単一ファイル（JSXとして動くことを優先。TypeScript構文なし）
- 依存：React（ランタイム想定）、CSSはTailwindクラス（環境依存）
- ストレージ：localStorage

## 7. 成果物
- 単一ファイルUI（index.tsx相当）
- 引き継ぎ用ドキュメント一式（本パック）
